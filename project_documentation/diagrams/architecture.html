Architecture Overview

Technology stack and architectural decisions at a glance

Design Decisions

LangGraph over plain functions: state machine makes branching and memory explicit; easier to extend sub-graphs independently
Each persona graph is compiled separately — adding a 5th persona is one new file
Hybrid RAG (FAISS + BM25): dense alone misses exact keyword matches; BM25 catches those; RRF merges without threshold tuning
Rule-based eligibility checker: LLM hallucinated income thresholds; deterministic rules are verifiable and never wrong
SSE over WebSockets: simpler server (no connection state), works through any CDN/proxy, browser EventSource handles reconnect

Scalability & Resilience

Circuit Breaker (per-provider): if Groq rate-limits, falls over to Ollama, then Anthropic — no single point of LLM failure
FAISS snapshots (5 rolling): admin mistakes recoverable by rolling back up to 5 states
Soft-delete + compaction: chunk deletions don't rebuild immediately; compaction only when ≥30% waste
TTL indexes on tokens/sessions: no cron job needed; MongoDB auto-evicts expired data
Async everywhere: Motor + asyncio.gather for concurrent web search; no blocking calls on event loop

Security Model

JWT HS256 + JTI blacklist: revocation on logout is explicit — no need to wait for expiry
Separate ADMIN_KEY: admin routes don't go through JWT flow — admin credentials never stored in MongoDB
Pydantic at every boundary: all request bodies validated before any handler runs
Rate limiting per user: keyed on JWT sub (not just IP) so authenticated users get fair throttling
bcrypt for passwords: cost factor 12 by default; no plaintext ever stored
